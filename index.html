Frequently Asked Questions

Find answers to common questions about our e-invoicing platform.

E-invoicing is the exchange of invoice documents between a supplier and a buyer in an integrated electronic format. Unlike PDFs or scanned documents, e-invoices are structured digital documents (like UBL XML) that can be processed automatically by accounting systems. In Australia, e-invoicing follows the PEPPOL framework and helps businesses reduce processing time and errors.

View our API documentation for more details

Our platform integrates directly with Reckon One's API to create and manage invoices. After connecting your Reckon One account, you can create invoices either manually or by uploading CSV/Excel files. The platform then automatically converts these invoices to UBL XML format for e-invoicing compliance. The integration requires your Reckon One credentials and bookId to access your business data.

Reckon One API UBL XML Conversion Automated Workflow

Our platform automatically validates invoices against Australian e-invoicing standards using the ESS Validator. The validation process checks for:

  • XML wellformedness
  • Schema validation against UBL 2.1 XSD
  • Business rules validation (EN16931)
  • PEPPOL rules validation (Australia-specific)

If validation fails, you'll receive detailed error messages to help you fix the issues. Once validated, invoices can be sent directly to recipients.

After an invoice has been validated, our platform offers two methods to send it to your customers:

Email

Send invoices via email with both PDF and UBL XML attachments. Recipients can view the PDF and process the UBL XML if their systems support it.

PEPPOL Network

Send invoices directly to recipients on the PEPPOL network using their business identifier (typically their ABN with a prefix).

Both methods are accessible directly from the validation results page, making the process seamless and efficient.

E-Invoicing Platform Documentation

This documentation outlines the business process flow and integration details for our e-invoicing platform, which helps SMEs create, validate, and send e-invoices that comply with Australian standards.

Platform Overview

Our e-invoicing platform integrates with Reckon One to provide a seamless e-invoicing solution for Australian SMEs. The platform allows you to:

  • Create invoices manually or by uploading CSV/Excel files

  • Convert invoices to UBL XML format compliant with Australian standards

  • Validate invoices against Australian e-invoicing standards using the ESS Validator

  • Send validated invoices via email or the PEPPOL network

Business Process Flow

Our e-invoicing platform follows a streamlined process to create, validate, and send e-invoices.

1. Invoice Creation
CSV/Excel/Manual Entry
2. UBL XML Conversion
Australian Standard Format
3. Invoice Validation
ESS Validator
4. Invoice Sending
Email or PEPPOL Network

Process Description

1. Invoice Creation

Create invoices using one of two methods:

  • Upload a CSV or Excel file with invoice data
  • Manually enter invoice details through our form

The invoice data is sent to the Reckon One API to create a standard invoice in their format.

2. UBL XML Conversion

After creating the invoice in Reckon One, our platform automatically converts it to UBL XML format, which is the standard required for Australian e-invoicing. This conversion maps all relevant fields to their corresponding UBL XML elements.

3. Invoice Validation

The UBL XML invoice is validated against Australian e-invoicing standards using the ESS Validator. This validation includes:

  • XML wellformedness checking
  • Schema validation against UBL 2.1 XSD
  • Business rules validation (EN16931)
  • PEPPOL rules validation (Australia-specific)

4. Invoice Sending

Once validated, the invoice can be sent using one of two methods:

  • Email: Send via email with PDF and UBL XML attachments
  • PEPPOL Network: Send directly to recipients on the PEPPOL network

Reckon One Integration

Our platform integrates with Reckon One to create and manage invoices. This section explains how the integration works.

Authentication

To connect with Reckon One, you need to authenticate using your Reckon credentials. Our platform securely stores your connection details and uses them to interact with the Reckon One API.

Book Selection

Reckon One organizes data into "books." After authentication, you'll need to select which book to use for your e-invoices. The bookId is a crucial parameter used in all API calls to Reckon One.

Important Note

All API calls to Reckon One require a bookId parameter. This identifies which set of business data to work with. You can find your available books in the Reckon One dashboard or through our platform's settings page.

API Endpoints Used

Endpoint Description
GET /{bookId}/invoices Retrieves a list of invoices from the specified book
POST /{bookId}/invoices Creates a new invoice in the specified book
GET /{bookId}/invoices/{id} Retrieves a specific invoice by ID
GET /{bookId}/contacts Retrieves customer contacts for invoice creation
GET /{bookId}/tax-rates Retrieves available tax rates for invoice line items

Invoice Creation

Our platform provides two methods for creating invoices: manual entry and CSV/Excel upload.

Manual Invoice Creation

The manual invoice form allows you to create invoices by entering all details directly. The form includes fields for:

  • Customer information (name, ABN, address, contact details)
  • Invoice details (number, date, due date, reference)
  • Line items (description, quantity, unit price, tax)
  • Notes and payment terms

When you submit the form, our platform creates the invoice in Reckon One using the following API call:

POST /{bookId}/invoices

UBL Conversion

After creating an invoice in Reckon One, our platform automatically converts it to UBL XML format, which is required for Australian e-invoicing.

Conversion Process

The conversion process maps Reckon One invoice fields to their corresponding UBL XML elements. This includes:

  • Invoice header information (ID, issue date, due date)
  • Supplier details (name, ABN, address, contact information)
  • Customer details (name, ABN, address, contact information)
  • Line items (description, quantity, unit price, tax)
  • Payment terms and instructions

UBL XML Structure

The UBL XML follows the structure required by the Australian e-invoicing standard, which is based on the PEPPOL BIS Billing 3.0 specification.

Validation

Our platform validates UBL XML invoices against Australian e-invoicing standards using the ESS Validator.

Validation Process

The validation process includes several checks to ensure the invoice complies with Australian e-invoicing standards:

  • XML Wellformedness: Checks that the XML is properly formed
  • Schema Validation: Validates against the UBL 2.1 XML Schema
  • Business Rules: Checks compliance with EN16931 business rules
  • PEPPOL Rules: Validates against Australia-specific PEPPOL rules

Validation Results

After validation, our platform displays the results, including any errors or warnings that need to be addressed. If the invoice passes validation, it's ready to be sent.

Successful Validation

Invoice is valid

All validation checks passed

  • XML is well-formed
  • Schema validation passed
  • Business rules validation passed
  • PEPPOL rules validation passed

Failed Validation

Invoice is invalid

Validation errors found

  • XML is well-formed
  • Schema validation passed
  • Business rules validation failed
  • PEPPOL rules validation failed
Error Details
  • Missing mandatory field: Buyer reference
  • Invalid tax category: Must be one of S, Z, E, AE, K, G, O, L, M

Sending Invoices

After an invoice has been validated, you can send it to your customer via email or the PEPPOL network.

Email Sending

When sending via email, our platform:

  • Generates a PDF version of the invoice
  • Attaches both the PDF and UBL XML files to the email
  • Sends the email to the customer's email address
  • Updates the invoice status in Reckon One

Email Template

The default email template includes:

  • Your company name and logo
  • Invoice number and amount
  • Due date and payment instructions
  • PDF and UBL XML attachments

You can customize the email message before sending.

PEPPOL Sending

When sending via the PEPPOL network, our platform:

  • Transmits the UBL XML invoice directly to the recipient's PEPPOL access point
  • Provides a transmission receipt with a unique identifier
  • Updates the invoice status in Reckon One

PEPPOL Requirements

To send via PEPPOL, you need the recipient's:

  • PEPPOL Participant ID (usually their ABN with a prefix)
  • Document type identifier (for invoices: urn:oasis:names:specification:ubl:schema:xsd:Invoice-2)
  • Process identifier (for invoices: urn:fdc:peppol.eu:2017:poacc:billing:01:1.0)

Our platform can look up the recipient's PEPPOL details using their ABN.

Frequently Asked Questions

What is UBL XML?

Universal Business Language (UBL) XML is a standardized XML format for business documents, including invoices. It's the format used for e-invoicing in Australia and many other countries.

What is the PEPPOL network?

PEPPOL (Pan-European Public Procurement Online) is a network that enables businesses to exchange electronic documents, including e-invoices, in a standardized way. Australia has adopted PEPPOL for e-invoicing.

Do I need to register for PEPPOL?

No, our platform handles the PEPPOL connection for you. You only need to register on our platform and connect your Reckon One account.

Can I still send PDF invoices?

Yes, when sending invoices by email, our platform automatically attaches a PDF version along with the UBL XML file.

What if my customer doesn't support e-invoicing?

You can still send the invoice by email with a PDF attachment. The customer can view and process the PDF invoice as usual.

How do I know if my invoice was delivered?

Our platform provides delivery status updates for both email and PEPPOL transmissions. You can check the status of your invoices on the dashboard.

What is the bookId in Reckon One?

The bookId is a unique identifier for your business data in Reckon One. It's required for all API calls to Reckon One. You can find your bookId in the Reckon One dashboard or through our platform's settings page.
